# 基本原理
   1. 发消息是发送到“交换器”，“交换器”根据交换器类型决定如何发往与它绑定的队列
   2. 创建一个队列，这个队列会默认绑定一个“空字符串”的交换器
   3. 监听同一个队列的多个应用会负载竞争地争抢消费此队列的消息（即一个队列的一条消息只可能被监听此队列的一个应用服务器消费）


# 广播策略
   1. 如果想发送一条消息，让此消息广播到每台服务器，那么可以程序启动时声明一个唯一队列名，且共同绑定一个交换器（此交换器是广播类型）
   2. 发送信息时对此交换器发送信息，就广播到每一个队列，由于每个队列各自有一台服务器监听，所以每台服务器可以接收到发送的同一条消息
